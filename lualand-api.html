LuaLand.RSL API v1.0

LuaLand.RSL API

Public read-only API for accessing scripts, executors, rbxm files and user data from LuaLand.RSL. Requires an API key to use.

Base URL https://zcioapswxzqttijygctt.supabase.co/rest/v1
Authentication

All requests require an API Key in the request header. Contact LuaLandHub on the platform to request a key.

Add your API key to every request header:
apikey: YOUR_API_KEY
Authorization: Bearer YOUR_API_KEY
Require Scripts
GET /posts Fetch all require scripts

Returns a list of all published require scripts.

Query Parameters

ParameterTypeDescription
typeoptionalstringFilter by type: Abilities, Guis, Spawner, Jumpscares, Tools
creatoroptionalstringFilter by uploader username
limitoptionalnumberMax results (default: 50, max: 100)
offsetoptionalnumberPagination offset

Example Request

GET /posts?type=Abilities&limit=10
apikey: YOUR_API_KEY
Authorization: Bearer YOUR_API_KEY

Response Schema

iduuidUnique post ID
titlestringScript name
descriptionstringScript description
codestringThe actual script code
typestringCategory type
tagsarrayList of tags
creatorstringUploader username
script_creatorstringActual script creator name
thumbnail_urlstringThumbnail image URL
likesnumberTotal likes
created_attimestampUpload date
GET /posts?title=ilike.*{query}* Search scripts

Search require scripts by title using partial match.

Example Request

GET /posts?title=ilike.*kill*&select=id,title,type,creator
apikey: YOUR_API_KEY
Authorization: Bearer YOUR_API_KEY

Rbxm Files
GET /rbxm_posts Fetch all rbxm files

Returns all published rbxm files. Note: file_url contains base64 data for download.

Query Parameters

ParameterTypeDescription
file_typeoptionalstringRbxm or Rbxmx
selectoptionalstringComma-separated columns (exclude file_url for faster response)

Example Request — exclude file data for listing

GET /rbxm_posts?select=id,name,description,file_type,creator,tags,created_at
apikey: YOUR_API_KEY
Authorization: Bearer YOUR_API_KEY
Users
GET /users?username=eq.{username} Fetch user profile

Returns public profile data of a user. Sensitive fields (password, protect_me, session_token) are never exposed.

Example Request

GET /users?username=eq.LuaLandHub&select=username,display_name,bio,avatar_url,followers,verified
apikey: YOUR_API_KEY
Authorization: Bearer YOUR_API_KEY

Safe Response Fields

usernamestringUnique username

Code Examples
JS JavaScript / Fetch Browser or Node.js 
const API_KEY = 'YOUR_API_KEY';
const BASE = 'https://zcioapswxzqttijygctt.supabase.co/rest/v1';

async function getScripts(type = null) {
  let url = `${BASE}/posts?select=*&order=created_at.desc`;
  if (type) url += `&type=eq.${type}`;

  const res = await fetch(url, {
    headers: {
      'apikey': API_KEY,
      'Authorization': `Bearer ${API_KEY}`
    }
  });
  return await res.json();
}

// Usage
const scripts = await getScripts('Abilities');
console.log(scripts);
LUA Roblox / HttpService Use in Roblox Studio 
local HttpService = game:GetService("HttpService")
local API_KEY = "YOUR_API_KEY"
local BASE = "https://zcioapswxzqttijygctt.supabase.co/rest/v1"

local function getScripts(scriptType)
    local url = BASE .. "/posts?select=title,code,type&order=created_at.desc"
    if scriptType then
        url = url .. "&type=eq." .. scriptType
    end

    local response = HttpService:RequestAsync({
        Url = url,
        Method = "GET",
        Headers = {
            ["apikey"] = API_KEY,
            ["Authorization"] = "Bearer " .. API_KEY
        }
    })

    if response.Success then
        return HttpService:JSONDecode(response.Body)
    end
    return nil
end

local scripts = getScripts("Abilities")
print(scripts)
Rate Limits & Rules
Read-only — GET requests only. POST, PATCH, DELETE are blocked.
Max 100 results per request. Use limit and offset for pagination.
Do not expose your API key publicly. Keep it server-side or in private scripts.
Abuse will result in key revocation. Do not spam requests or use for unauthorized purposes.
